# MediFirst 게시판 UI 개선 계획서

## 📋 참고 이미지 분석 결과

### 현재 상태 vs 참고 이미지 비교

#### 1. 게시글 목록 (List View)
**참고 이미지 특징:**
- ✅ 썸네일 이미지 (왼쪽)
- ✅ 제목 + 댓글 수 아이콘 (제목 옆에 작은 아이콘)
- ✅ 좋아요 수 (👍 아이콘)
- ✅ 카테고리 배지 ("일반" 등)
- ✅ 작성자명
- ✅ 조회수 (👁️ 아이콘)
- ✅ 시간 (상대 시간: "43분전", "1시간전")
- ✅ 한 줄 레이아웃 (깔끔한 정렬)

**현재 MediFirst:**
- ✅ 제목, 댓글 수, 조회수, 작성자, 시간 표시됨
- ❌ 썸네일 이미지 없음
- ❌ 좋아요 수 표시 안됨 (0으로 고정)
- ⚠️ 레이아웃은 비슷하지만 시각적 개선 여지 있음

#### 2. 게시글 상세 페이지
**참고 이미지 특징:**
- ✅ 상단 네비게이션 (이전글 ← / 목록 / 다음글 →)
- ✅ 제목 + 카테고리 배지
- ✅ 작성자 프로필 이미지 (원형 아바타)
- ✅ 작성자명 + 시간 + 조회수 (한 줄)
- ✅ 본문 내용
- ✅ 댓글 섹션 (댓글 작성 폼이 상단)
- ✅ 하단에 "전체글" 목록

**현재 MediFirst:**
- ✅ 목록으로 버튼 있음
- ❌ 이전글/다음글 네비게이션 없음
- ❌ 작성자 프로필 이미지 없음
- ✅ 댓글 섹션 있음
- ✅ 하단 관련 게시글 목록 있음

#### 3. 추가 기능
**참고 이미지 특징:**
- ✅ 검색 기능 (제목 검색)
- ✅ 페이지네이션 (1, 2, 3, 4, 5, 다음 →)
- ✅ 썸네일 이미지 지원

**현재 MediFirst:**
- ❌ 검색 기능 없음
- ✅ 페이지네이션 있음 (하지만 스타일 개선 필요)
- ❌ 썸네일 이미지 없음

## 🎯 개선 방향 제안

### 1. 게시글 목록 개선 (우선순위: 높음)

#### A. 썸네일 이미지 추가
```typescript
// PostListRow 컴포넌트에 썸네일 추가
interface PostListRowProps {
  // ... 기존 props
  thumbnailUrl?: string | null  // 새로 추가
}
```

**구현 방법:**
- `posts` 테이블에 `thumbnail_url` 컬럼 추가 (nullable)
- 게시글 작성 시 이미지 업로드 기능 추가 (선택사항)
- 썸네일이 없으면 기본 이미지 또는 아이콘 표시

**레이아웃:**
```
[썸네일] [필독] [카테고리] 제목 💬3  |  작성자  👁️62  43분전  👍3
```

#### B. 좋아요 기능 활성화
- 현재 좋아요 수가 0으로 고정되어 있음
- 좋아요 기능 구현 필요 (별도 테이블 또는 필드)

#### C. 시각적 개선
- 아이콘 크기 및 간격 조정
- 호버 효과 강화
- 모바일 반응형 개선

---

### 2. 게시글 상세 페이지 개선 (우선순위: 높음)

#### A. 상단 네비게이션 추가
```
[← 이전글]  [☰ 목록]  [다음글 →]
```

**기능:**
- 같은 카테고리/게시판의 이전/다음 게시글 이동
- 목록 버튼 (현재 "목록으로" 버튼과 통합)

#### B. 작성자 프로필 이미지 추가
- 원형 아바타 이미지 표시
- `profiles` 테이블에 `avatar_url` 컬럼 추가 필요
- 기본 아바타 (이니셜 또는 기본 이미지)

**레이아웃:**
```
[프로필 이미지] 작성자명  ·  12시간전  ·  조회 121  [⋯]
```

#### C. 댓글 섹션 개선
- 댓글 작성 폼을 상단으로 이동 (현재는 하단)
- 댓글 목록을 작성 폼 아래에 표시

---

### 3. 검색 기능 추가 (우선순위: 중간)

#### A. 검색 바 추가
**위치:** 게시판 목록 상단 또는 하단

**기능:**
- 제목 검색
- 작성자 검색 (선택사항)
- 검색어 하이라이트

**UI:**
```
[제목 ▼] [검색어 입력...] [🔍 검색]
```

---

### 4. 페이지네이션 개선 (우선순위: 낮음)

**현재:** 이전/다음 버튼 + 페이지 번호 (최대 5개)

**개선:**
- 페이지 번호 스타일 개선
- "다음 →" 버튼 스타일 통일
- 모바일에서 간소화

---

### 5. 썸네일 이미지 시스템 (우선순위: 중간)

#### A. 이미지 업로드 기능
- 게시글 작성 시 이미지 첨부 (선택사항)
- Supabase Storage 활용
- 이미지 리사이징 (썸네일 자동 생성)

#### B. 기본 썸네일
- 이미지가 없을 경우:
  - 카테고리별 기본 이미지
  - 또는 텍스트 기반 썸네일 (제목 첫 글자)

## 🎨 디자인 가이드라인

### 색상
- 주요 액션: 티일/청록색 계열 (#14b8a6, #0d9488)
- 배지: 카테고리별 색상 구분
- 텍스트: 계층 구조 명확히

### 간격
- 게시글 간격: py-3 (현재) → py-4 (개선)
- 아이콘과 텍스트 간격: gap-1 → gap-1.5

### 아이콘
- 일관된 아이콘 크기: h-3 w-3 (작은 아이콘), h-4 w-4 (중간)
- 아이콘 색상: text-muted-foreground

### 반응형
- 모바일: 썸네일 숨김 또는 작게 표시
- 태블릿: 중간 크기
- 데스크톱: 전체 정보 표시

## 📝 데이터베이스 변경사항

### 1. posts 테이블
```sql
ALTER TABLE posts ADD COLUMN IF NOT EXISTS thumbnail_url TEXT;
```

### 2. profiles 테이블
```sql
ALTER TABLE profiles ADD COLUMN IF NOT EXISTS avatar_url TEXT;
```

### 3. post_likes 테이블 (선택사항)
```sql
CREATE TABLE IF NOT EXISTS post_likes (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  post_id UUID NOT NULL REFERENCES posts(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(post_id, user_id)
);
```
